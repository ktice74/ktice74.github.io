Arrays of Insight: Hashing out Arrays and Hashes in Ruby

Arrays and hashes are just a list on your computer, or more specifically, they are indexed collections. Both store collections of objects and are accessible using a key. The reason that you would want to use an array or hash is because variables can only hold one item at a time. Arrays and hashes can hold multiple items and the list can grow over time.

For arrays, the keys are integers, as each object in the list is automatically assigned a place holder value (or in Ruby speak, an index) which starts at zero.

Let’s go ahead and create a list. >> data = [“mishu”, 3, 0.745, true, (0, 3)] Arrays can be heterogenous, meaning that all of the values do not have to be the same data types. The above is a valid array, and you will not receive an error even though the values are respectively, a string, integer, float, boolean value and then another array.

Arrays

Each value within the array has a placeholder (or in Ruby-speak, indexes), which start with 0. So if we were to ask ruby: data[0] it would return “mishu” because the string “mishu” is in the first placeholder. We can also see how many total values are contained in the array by asking >> data.size which returns => 5.

While the values in an array have the keys of automatically assigned indexes, hashes on the other hand allow you to assign each value with their own unique key.

As you can imagine, arrays are easier to build and keep track of, while hashes allow for more flexibility.

Hashes

Take a look at this hash (and notice that we’re using a squiggly bracket instead of a square bracket that are assigned to arrays. >> home_teams = { ‘Brewers’ => ‘baseball’, ‘Packers’ => ‘football’, ‘Bucks’ => ‘basketball’, ‘Admirals’ => ‘hockey’, ‘Wave’ => ‘soccer' }

In this hash, I’ve assigned the values of each (Milwaukee area) home sports team and assigned their keys to be the sports that they represent.

So if I wanted to know the home_team [‘football’], Ruby would tell me that the value would be ‘Packers’.

I could then create a separate hash for adopted_hometeams and list all of the Bay Area sports teams: >> adopted_hometeams = { ‘Giants’ => ‘baseball’, ‘49ers’ => ‘football’, ‘Warriors’ => ‘basketball’, ’Sharks’ => ‘hockey’, ‘Earthquakes’ => ‘soccer’, }

If I were to ask for adopted_hometeams [‘football’], ruby would return ‘49ers’. Since there are individualized local variables for each array, we don’t have to worry about having the same key names within each hash.
