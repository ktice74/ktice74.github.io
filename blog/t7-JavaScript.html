After spending a few weeks getting familiar with Ruby, this week we were introduced to JavaScript. Just like two different human languages, JavaScript and Ruby look and behave similarly but where they differ is in their syntax.

In this entry, I will describe how JavaScript Objects and Ruby hashes differ.

Let's go ahead and build a JavaScript object:

JavaScript

var Simpsons = {};
Simpsons.dad = "Homer"
Simpsons.mom = "Marge"

is the same as:

var Simpsons = {
dad: "Homer",
mom: "Marge",
};

If we would want to call the object property for "name", we would do so by calling:
Simpsons["dad"] = "Homer"

This is all very similar to the setup of a hash in Ruby.

While there are a couple of different ways to create hashes in Ruby, lets use this the Hash.new method as an example:

Simpsons = Hash.new
Simpsons = {}
Simpsons["dad", "Homer", "mom", "Marge"]
Simpsons = {"dad" => "Homer", "mom" => "Marge"}

This looks very similar to the var Simpsons value.

As you can see, the end result is the same. However, lets "peak under the hood" to see exactly what is occuring when we are making these calls.

In JavaScript, we are creating an object, called "object." In order to create, this object, we are declaring the variable name and assigning an empty value.

var object = {}
var object = {
name: value
}

And therefore...
var object.name = value
--> We are looking for the name property of the object.

h = Hash.new
h = {}
h = {key => "key value pair"}
h[key] = "key value pair"
--> We are looking for the "key value pair" of the key "key"
